A fully parameterized, production-quality **L2 Cache Controller** targeting
multi-core SoC designs. Implements MESI coherency, AXI4/ACE interfaces,
pseudo-LRU replacement, SECDED ECC, and a 16-entry MSHR — paired with a
complete UVM-1.2 verification environment, JasperGold formal properties,
Synopsys DC synthesis scripts, and a Python regression runner.

## Parameters

```systemverilog
module l2_cache_top #(
  parameter int unsigned CACHE_SIZE_KB = 256,   // 64 / 256 / 512 / 1024
  parameter int unsigned WAYS          = 4,     // 2 / 4 / 8 / 16
  parameter int unsigned LINE_SIZE_B   = 64,    // 32 or 64 bytes
  parameter int unsigned ADDR_WIDTH    = 40,    // physical address bits
  parameter int unsigned DATA_WIDTH    = 64,    // AXI data bus width
  parameter int unsigned MSHR_DEPTH    = 16,    // outstanding misses
  parameter int unsigned NUM_BANKS     = 4,     // data SRAM banks
  parameter int unsigned ID_WIDTH      = 8      // AXI ID width
)
```

All structural parameters (`NUM_SETS`, `INDEX_BITS`, `TAG_BITS`, `OFFSET_BITS`,
`WORDS_PER_LINE`, `LRU_BITS`) are derived automatically via `localparam`.

| 1 MB 8-way | 1 MB | 2048 | 8 | 23b | 11b | 6b |
| 64 KB 2-way | 64 KB | 512 | 2 | 25b | 9b | 6b |

## Cache Pipeline (2-stage hit path)

| Cycle | Action |
|---|---|
| 0 | AXI address received; set index + tag extracted; tag RAM + data RAM address driven |
| 1 | Tag comparison completes; hit/miss resolved; PLRU updated |
| HIT | Data MUX → AXI R channel; total read latency = **2 cycles** |
| MISS | MSHR allocated; AXI fill issued; fill latency = **20–200 cycles** (L3/DRAM) |

---

## MESI Coherency Protocol

| State | Description |
|---|---|
| **I** (Invalid) | Line not present |
| **S** (Shared) | Clean, may exist in peer caches |
| **E** (Exclusive) | Clean, only in this cache |
| **M** (Modified) | Dirty, only in this cache — must write back before sharing |

**Key transition rules:**

- `E → M`: silent (no bus transaction needed)
- `S → M`: broadcast upgrade (CleanUnique on ACE AW channel)
- `M → S`: dirty data forwarded on CD channel (PassDirty=1 in CRRESP)
- `M → I`: write-back to memory + invalidate

**AXI-ACE snoop channels:**

| Channel | Direction | Purpose |
|---|---|---|
| AC | Interconnect → L2 | Snoop address + type |
| CR | L2 → Interconnect | 5-bit response (DataTransfer, PassDirty, IsShared, WasUnique, Error) |
| CD | L2 → Interconnect | Dirty data forwarding (when PassDirty=1) |

## Running Simulation

### Prerequisites

```
Synopsys VCS 2023.03+  or  Cadence Xcelium 23.09+
UVM 1.2 (bundled or Accellera)
Python 3.8+  with  pyyaml  (pip install pyyaml)
```

### Single test — VCS

```bash
# Compile
vcs -full64 -sverilog -f sim/vcs/vcs_filelist.f \
    -o sim/vcs/l2_sim -Mdir sim/vcs/csrc

# Run smoke test
./sim/vcs/l2_sim \
    +UVM_TESTNAME=l2_smoke_read_test \
    +ntb_random_seed=12345 \
    +UVM_VERBOSITY=UVM_MEDIUM \
    -l reports/regression/smoke.log
```

### Single test — Xcelium

```bash
xmvlog -64bit -sv -f sim/xcelium/xm_filelist.f
xmsim -64bit +UVM_TESTNAME=l2_smoke_read_test \
      +ntb_random_seed=12345 l2_cache_tb_top \
      -l reports/regression/smoke.log
```

### Full regression (Python, parallel)

```bash
# 8-core regression — all tests, 3 seeds each
python3 scripts/regression/run_regression.py \
    --plan scripts/regression/test_plan.yaml \
    --tool vcs --jobs 8

# CI gate (smoke + directed, 1 seed, fast)
python3 scripts/regression/run_regression.py --tag ci --seeds 1

# Nightly (all tests × 10 seeds)
python3 scripts/regression/run_regression.py --seeds 10 --jobs 16

# Single named test with specific seed
python3 scripts/regression/run_regression.py \
    --test l2_mesi_snoop_read_shared_test --seed 42
```

Output: `reports/regression/regression_report.html`

## RTL Coding Style

All RTL follows these conventions (enforced by SpyGlass):

```systemverilog
// ✅ Sequential logic
always_ff @(posedge clk or negedge rst_n) begin
  if (!rst_n) state <= IDLE;
  else        state <= next_state;
end

// ✅ Combinational — default assignment prevents latches
always_comb begin
  next_state = state;          // default
  unique case (state)
    IDLE: if (req) next_state = ACTIVE;
    default: next_state = IDLE;
  endcase
end

// ✅ Use logic everywhere (not reg or wire)
logic [TAG_BITS-1:0] tag_compare;

// ✅ Import package explicitly
import l2_cache_pkg::mesi_state_t;
```

## Design for Test (DFT)

The L2 cache ships with a complete DFT infrastructure in the `dft/` folder.

### DFT folder structure

```
dft/
├── rtl/
│   ├── l2_cache_dft_top.sv    ← DFT wrapper: scan ports, clock mux, BIST hook
│   ├── l2_bist_ctrl.sv        ← March-C BIST controller for SRAM macros
│   └── l2_scan_wrapper.sv     ← ICG bypass cell, scan mux, observation point
├── atpg/
│   ├── run_atpg.tcl           ← TetraMAX ATPG: stuck-at + transition campaign
│   └── gls_atpg_tb.v          ← Gate-level simulation testbench for patterns
├── patterns/
│   └── l2_scan_continuity.stil← Reference STIL scan continuity patterns
└── docs/
    └── dft_architecture.md    ← Full DFT architecture specification
```

### Scan chain architecture

| Chain | Contents | FFs (est.) |
|---|---|---|
| 0 | Request pipeline + tag array pipeline regs | ~2900 |
| 1 | LRU PLRU state (512×3b) + MSHR entry state FFs | ~2920 |
| 2 | Coherency FSM state + snoop capture regs | ~2910 |
| 3 | AXI master state + performance counters | ~2900 |

**Total: ~11 648 FFs across 4 chains**  
Scan cell style: **Multiplexed flip-flop (MUX-D)**  
ICG bypass: all ICG cells replaced with `l2_icg_scan_bypass` (scan-transparent)

### BIST (Built-In Self-Test)

The `l2_bist_ctrl` implements the **March-C algorithm** for SRAM self-test:

```
M0: ↑(w0)      — write 0, ascending
M1: ↑(r0, w1)  — read 0, write 1, ascending
M2: ↑(r1, w0)  — read 1, write 0, ascending
M3: ↓(r0, w1)  — read 0, write 1, descending
M4: ↓(r1, w0)  — read 1, write 0, descending
M5: ↓(r0)      — read 0, descending
```

Fault coverage: stuck-at, transition, coupling, address decoder faults.  
BIST time: **~7.2 ms** for 256 KB at 100 MHz test clock.  
Results: `bist_pass` (pass/fail) + `bist_fail_map[15:0]` (per-macro diagnostic).
